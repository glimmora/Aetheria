# Mythral — Mobile UI/UX Redesign Plan

## Context

The pixel-art vertical slice (Island 1) is complete. Now we improve **mobile play**.
The codebase already has an extensive responsive stylesheet (`client/src/styles/game.css`
lines ~2068–2459) with `isMobile` detection (`useGame.js:67`), a top HUD, a bottom
skill bar, and full-screen modal windows. **But mobile UX is poor in practice** because
the styling exists while the *interaction model* is still desktop-first:

- **No on-screen movement controls.** Movement is keyboard (WASD) on desktop and
  tap-to-move on click. On a phone there is **no virtual joystick / D-pad**, so the
  player can only tap-to-move — awkward for combat, dodging, and precise navigation.
- **Crowded top HUD.** 8 icon buttons + gold + online-count + menu button fight for
  space; `.hud-location` and `.hud-combat-log` are simply hidden on phones (lost info).
- **Modal windows block the game** with no tap-outside-to-close, no swipe-to-dismiss,
  and a small X — poor on touch.
- **No "action menu" pattern** — the 8 HUD buttons occupy permanent screen space
  instead of collapsing behind the menu button.
- **Tap ambiguity** — `handleGameMouseDown` in `GameScreen.jsx:146` treats every tap
  identically (move vs interact vs open dialog); no long-press / no distinction between
  "walk here" and "talk to NPC".
- **No safe-area / notch handling** (`env(safe-area-inset-*)`), and landscape phone
  hides minimap + chat entirely (`game.css:2453`).
- Auth/char-select are sized but not thumb-optimized (buttons reachable, but no
  large touch targets / no mobile-specific flow).

**Goal:** Make Mythral genuinely playable and pleasant on phones (portrait + landscape)
without changing any gameplay, networking, or backend logic. Desktop experience must
not regress.

---

## Design Principles

1. **Thumb-zone layout** — primary controls (movement, attack, skills) in easy-reach
   bottom corners; secondary/menus behind a single button.
2. **Progressive disclosure** — collapse the 8 HUD buttons into a slide-up action
   drawer opened from the menu button; surface only combat-critical info by default.
3. **Touch-native windows** — tap-outside-to-close, swipe-down-to-dismiss, large
   close targets, never fully opaque (game peeks through).
4. **Virtual joystick** — floating, drag-anywhere left thumb zone; emits `sendMove(dx,dy)`
   continuously (server already supports single-step MOVE — `useGame.js:443`).
5. **Non-blocking HUD** — HP/MP/XP always visible and compact; everything else on demand.
6. **Safe areas** — respect notches with `env(safe-area-inset-*)`.
7. **No new deps, no backend changes.** Pure CSS + small React components/hooks.

## Implementation Plan

### 1. Virtual Joystick (movement) — `client/src/components/ui/Joystick.jsx` (new)
- A floating control rendered only when `isMobile`. Left-bottom thumb zone.
- Pointer events (`pointerdown/move/up`) compute an 8-direction (or analog) vector;
  while held, call `sendMove(dx,dy)` on a ~120ms interval (reuse existing throttle).
  Release calls `sendMove(0,0)` / `stopMoveTo()`.
- Visual: a pixel-art base + knob drawn with existing `fx`/UI sprite language
  (or pure CSS circle with gold rim to match `pixel-ui.css`).
- Wire into `GameScreen.jsx`: render `<Joystick onMove={sendMove} />` inside the
  game screen when `isMobile`.

### 2. Attack / interact buttons — bottom-right cluster
- A large "Attack" button (auto-targets nearest monster via existing
  `targetMonster` logic in `GameScreen.jsx:212`) + an "Interact" button that triggers
  the nearest NPC dialog (mirrors keyboard flow). These live bottom-right (thumb zone),
  distinct from the joystick (bottom-left).
- Tap-to-move on the map still works (joystick is an *additional* primary control).

### 3. Collapsible Action Drawer (the 8 HUD buttons)
- Keep `hud-buttons` but on mobile hide them from the top bar; render them inside a
  slide-up panel triggered by the existing `game-menu-btn` (☰). Each entry becomes a
  large, labeled, thumb-friendly row (icon + label), matching the fantasy pixel theme.
- The top bar on mobile shows only: portrait + name/class, HP/MP/XP bars, gold,
  online count — and the menu button. This frees vertical space and stops overlap with
  the skill bar.

### 4. Skill bar — mobile refinement
- Keep the 6-slot bar but ensure it sits **above** the joystick/attack cluster with no
  overlap; reduce/cap its width, allow horizontal scroll if needed, and make slots
  larger tap targets (already 46px on phone — verify spacing vs joystick).
- Show skill icon + cooldown ring; drop the tiny name/cost text on phone (or show on
  long-press tooltip) to reduce clutter.

### 5. Touch-native Windows (`dialog-overlay`, `.mythral-window`)
- Add **tap-outside-to-close** (click on `.dialog-overlay` backdrop closes; stop
  propagation on the window) for all modal windows when `isMobile`.
- Add **swipe-down-to-dismiss** (pointer drag on header) — optional, lightweight.
- Make the close (×) button larger on touch; ensure window never covers 100% so the
  game peeks (already 96vw/92vh on phone — keep, add slight transparency to body).
- Ensure scrollable window bodies and sticky headers/footers (action buttons) so long
  lists (shop, quest log, inventory) are usable.

### 6. Layout / safe-area / landscape
- Add `env(safe-area-inset-*)` padding to `.game-screen`, HUD, skill bar, joystick,
  action drawer so content clears notches/home indicators.
- Landscape phone: instead of hiding minimap+chat (current `game.css:2453`), **relayout**
  them into slim side panels; keep joystick bottom-left, skills bottom-right.
- Prevent page scroll/zoom on mobile: `touch-action: none` on the game container,
  `user-scalable=no` already implied by viewport (verify `index.html` meta viewport).

### 7. Tap-intent disambiguation (`GameScreen.jsx`)
- Short tap on map = move (existing `handleTileClick`). 
- Tap on an entity (NPC/monster/player) already opens dialog/attacks — keep.
- Add a subtle **hold-to-interact** hint: long-press an NPC opens dialog even if far
  (instead of walking). Keep simple; primarily rely on the dedicated Interact button.

### 8. Auth & Char-Select mobile polish
- Verify `.auth-card` / `.char-card` use large inputs/buttons (already mostly there).
- Ensure the register/login form and class-select grid are thumb-friendly; add
  safe-area padding; confirm single-column layouts at 480px (already present) read well.
- Add a "play as guest/demo" is out of scope; just ensure the flow fits a phone.

### 9. Settings: "Mobile comfort" toggles (optional, low-risk)
- Add toggles to `settings`: `compactHud`, `leftHanded` (mirror joystick/attack sides),
  `joystickSize`. Persist via existing `updateSettings`. This gives users control
  without forcing a single layout. (Keep minimal — only if time permits; not required
  for the core fix.)

## Files to Touch (all additive / CSS + small components)

- `client/src/components/ui/Joystick.jsx` — **new** virtual joystick.
- `client/src/components/screens/GameScreen.jsx` — mount joystick + attack/interact
  buttons when `isMobile`; add tap-outside handling on windows.
- `client/src/components/ui/HUD.jsx` — optionally hide top `hud-buttons` on mobile
  (moved to drawer) — or leave markup and hide via CSS.
- `client/src/styles/game.css` — mobile blocks: joystick, attack cluster, action
  drawer, safe-area, landscape relayout, larger close targets, non-blocking windows.
- `client/src/styles/pixel-ui.css` — theme the new mobile controls to match fantasy
  pixel art (gold rims, parchment drawers).
- `client/index.html` — confirm viewport meta (`viewport-fit=cover` for safe areas).
- `client/src/hooks/useGame.js` — expose `sendMove` (already passed to GameScreen) and
  add `leftHanded`/`compactHud` settings if implementing §9.

**Backend / shared / gameplay:** unchanged.

## Verification

- `npm run build` succeeds; `npm run smoke` still 50/50 (no logic change).
- Manual mobile checks (Chromium devtools device emulation + real touch via pointer
  events, or the existing headless harness):
  - Joystick moves the player (verified via position change / server echo).
  - Attack button damages a nearby monster; Interact opens NPC dialog.
  - Menu button opens action drawer with all 8 functions; tapping one works; drawer
    closes on selection.
  - Opening Inventory/Shop/Quest windows on phone: tap outside closes; X is large;
    content scrolls.
  - No overlap between HUD bars, skill bar, joystick, and attack cluster in portrait
    (320×568, 375×667, 414×896) and landscape.
  - Safe-area: content not hidden under a simulated notch.
  - Desktop (≥1024px) layout unchanged / not regressed.
- Confirm no new console errors in the headless run.

## Risks / Notes
- Joystick must throttle emits to avoid flooding the 10Hz server / rate limits
  (`useGame.js` has rate limiting) — emit at ~100–150ms while held, matching server tick.
- Keep `isMobile` as the gate so desktop is fully untouched.
- The change is CSS + 1–2 small components; low risk of breaking gameplay.
