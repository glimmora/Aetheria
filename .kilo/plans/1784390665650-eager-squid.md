# Plan: Sempurnakan Proyek Mythral ("works 100%")

## Konteks
Mythral adalah game MMORPG browser multiplayer (React+Vite client, Node+Express+Socket.io server, data/logic di `shared/`). Proyek ini sudah sangat lengkap: 9 pulau, 113 NPC, 89 quest, 35+ monster (8 boss + 1 final boss), 4 class, 60+ item, auth JWT, persistensi JSON, anti-cheat, chat, leaderboard, inspect, dll. Commit terakhir mengklaim smoke test + E2E + functional audit lulus.

Tujuan: **pastikan proyek benar-benar berjalan 100%** — build bersih, semua test lulus, dan memperbaiki bug/inkonsistensi nyata yang ditemukan saat review.

## Hasil Review (kode sudah konsisten & solid)
Semua file inti sudah ditinjau: `server/{index,world,db,auth,security,serve-client}.js`, `shared/{protocol,combat,classes,items,monsters,islands,quests,inventory,tiles,islandGenerator}.js`, dan seluruh komponen client (`App`, `useGame`, `GameScreen`, `TileMap`, `HUD`, semua window UI, `Auth/CharSelect` screen, `pathfinding`, `ErrorBoundary`). Tidak ditemukan crash atau ketidakcocokan field antar client↔server yang fatal.

## Masalah Nyata yang Akan Diperbaiki

### 1. Inventaris tidak menegakkan batas slot (`MAX_INVENTORY_SLOTS`) secara konsisten
- `shared/inventory.js` → `addItemToInventory()` **tidak** mengecek `CONFIG.MAX_INVENTORY_SLOTS`.
- Di server, hanya jalur *monster loot* (`world.js:handleMonsterDeath`) yang mengecek batas; jalur **quest turn-in** dan **equip/unequip** (item kembali ke inventory) tidak mengecek batas.
- `shared/quests.js:turnInQuest` menggunakan `addItemToInventoryHelper` (duplikat privat) yang juga mengabaikan batas → item reward bisa melebihi kapasitas.
- **Perbaikan:** buat `addItemToInventory(itemId, qty)` menghormati `CONFIG.MAX_INVENTORY_SLOTS` (kembalikan `false`/abaikan jika penuh), gunakan fungsi shared yang sama di `quests.js` (hapus duplikat `addItemToInventoryHelper`), dan pastikan server menangani kasus inventory penuh saat menerima item (notifikasi "Inventory full" seperti pada loot). Ini menyelaraskan semua jalur penambahan item.

### 2. Verifikasi build & test yang sebenarnya (langkah paling penting)
Shell saat ini memblokir `node`/`npm`/`find` (aturan bash), jadi build/test belum dijalankan. Selama implementasi WAJIB:
- `npm install` (workspace client+server).
- `npm run build` (client) → pastikan sukses tanpa error.
- `npm run smoke` → target 50+ test lulus.
- Jalankan server (`node server/index.js` / `npm run dev:server`) lalu `node scripts/e2e-test.js` dan `node scripts/full-audit.js` → target lulus 100%.
- Jika ada test gagal, perbaiki akar masalahnya (bukan sekadar menghapus test).

### 3. Kecil / hardening (opsional, rendah risiko)
- `client/index.html` menyuntikkan `process = { env: {} }` sehingga konfigurasi `.env` di sisi client (PORT, TICK_RATE, dll) terabaikan. Ini aman karena client tidak melakukan tick dan `vite.config.js` sudah membaca env via `loadEnv` untuk proxy dev. Biarkan, tapi pastikan tidak ada asumsi client bahwa `CONFIG` dari `.env` berlaku.
- Pastikan `client/public` memuat aset yang direferensikan `index.html`: `/favicon.svg`, `/icon-192.png`, `/manifest.json`. Jika tidak ada, tambahkan stub sederhana agar tidak 404 (tidak fatal tapi rapi).
