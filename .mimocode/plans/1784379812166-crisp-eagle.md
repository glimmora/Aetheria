# Plan: Separate Frontend (12000) and Server (12400)

## Problem
Currently in production, the server serves the built client from port 12400 via SPA fallback. User wants frontend on port 12000, server on port 12400 — two separate processes.

Dev mode already works this way (vite on 12000, server on 12400). Only production mode needs fixing.

## Changes

### 1. Create `server/serve-client.js` — lightweight static file server
- Serves `client/dist/` on port 12000
- SPA fallback for client-side routing
- No dependencies beyond what's already installed (uses built-in `http` + `fs` + `path`)

### 2. Remove SPA fallback from `server/index.js`
- Delete the production static serving block (lines 464-472)
- Server only handles API + WebSocket on port 12400

### 3. Update `ecosystem.config.cjs`
- Add second PM2 app: `mythral-client` running `server/serve-client.js` on port 12000
- Main `mythral` app stays on port 12400

### 4. Update `scripts/prod.sh`
- Stop/start both PM2 processes
- Status shows both

### 5. Update `scripts/dev.sh`
- No changes needed (already correct: vite on 12000, server on 12400)

## Files to modify
| File | Action |
|------|--------|
| `server/serve-client.js` | CREATE — static file server on port 12000 |
| `server/index.js` | EDIT — remove SPA fallback block |
| `ecosystem.config.cjs` | EDIT — add mythral-client app |
| `scripts/prod.sh` | EDIT — handle both processes |
